## Conventions

| Item | Value |
|------|--------|
| **Protocol** | HTTPS in production |
| **JSON** | Request bodies: `Content-Type: application/json` |
| **CORS** | Server uses `cors()` — browser calls from other Vercel/static hosts are allowed |
| **Errors** | Most failures return JSON `{ "error": "message" }` unless noted (file downloads) |

## 1. Health check

**`GET {API_BASE}/health`**

No auth, no body.

### Response `200` — JSON

```json
{
  "status": "ok",
  "service": "AeroMaverick Chatbot API",
  "timestamp": "2026-03-26T12:00:00.000Z",
  "sessions": 0,
  "leads_file": "/path/or/tmp-file.xlsx",
  "database": "configured",
  "admin_panel": "open"
}
```

| Field | Meaning |
|-------|---------|
| `database` | `"configured"` if Postgres is wired; else `"off"` |
| `admin_panel` | `"auth_required"` if server has `ADMIN_API_KEY`; else `"open"` |

Use this to verify the deployment is reachable before wiring chat or admin.

## 2. Public config (bootstrap UI)

**`GET {API_BASE}/config`**

No auth. Use once at app load to discover API URL, feature flags, chatbot list.

### Response `200` — JSON

```json
{
  "publicApiUrl": "https://backend-admin-alpha.vercel.app",
  "port": 3000,
  "database": true,
  "adminAuthRequired": false,
  "openai": true,
  "chatbots": [
    { "slug": "aeromaverick", "label": "AeroMaverick" },
    { "slug": "safepay4u", "label": "Safepay4u" }
  ],
  "defaultBot": "aeromaverick",
  "adminShowAllBots": true
}
```

| Field | Meaning |
|-------|---------|
| `publicApiUrl` | Canonical API URL (from env or request host) |
| `database` | Whether DB env is set (not a live “ping”) |
| `adminAuthRequired` | If `true`, all `/admin/*` routes expect the admin key (see §5) |
| `openai` | Whether `OPENAI_API_KEY` is set (chat will fail if `false`) |
| `chatbots` | Allowed slugs for filters; list may be trimmed by server env |
| `defaultBot` | Slug used when chat request omits `bot` |

## 3. Chat (main integration)

**`POST {API_BASE}/chat`**

Maintains **conversation memory** per `session_id`. If you omit `session_id`, the server creates one and returns it — **store it** and send it on every subsequent message for the same user/thread.

### Request body — JSON

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `message` | string | **Yes** | User message (non-empty after trim). |
| `session_id` | string (UUID) | No | Omit on first message; reuse on follow-ups for same conversation. |
| `bot` | string | No | Workspace slug; invalid values fall back to server default (see §6). |

### Example — first message

```json
{
  "message": "I’m looking for a light jet under $3M.",
  "bot": "aeromaverick"
}
```

### Example — follow-up

```json
{
  "session_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "message": "I’m based in Texas.",
  "bot": "aeromaverick"
}
```

### Response `200` — JSON

```json
{
  "session_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "bot": "aeromaverick",
  "reply": "Assistant text shown to the user…",
  "lead_saved": false,
  "lead_data": null
}
```

| Field | Type | Description |
|-------|------|-------------|
| `session_id` | string | Always returned; persist client-side for the thread. |
| `bot` | string | Normalized slug used for this session. |
| `reply` | string | Assistant reply (lead JSON block stripped if model emitted one). |
| `lead_saved` | boolean | `true` if a lead was persisted (DB and/or Excel) this turn. |
| `lead_data` | object \| null | Present only when `lead_saved` is `true` — same shape as lead row (see §7). |

### Error responses — JSON

| Status | Body | When |
|--------|------|------|
| `400` | `{ "error": "message is required" }` | Missing/blank `message`. |
| `429` | `{ "error": "OpenAI rate limit reached. Please try again shortly." }` | Upstream rate limit. |
| `500` | `{ "error": "Invalid OpenAI API key." }` | OpenAI auth failure. |
| `500` | `{ "error": "Internal server error. Please try again." }` | Other server errors. |

## 4. Download all leads (Excel, legacy)

**`GET {API_BASE}/leads`**

Returns **`.xlsx`** binary (not JSON). No admin key (intended as a simple export; tighten on server if needed).

### Response `200`

- `Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- `Content-Disposition: attachment; filename="aeromaverick_leads.xlsx"`
- Body: Excel file stream

### Response `500` — JSON

```json
{ "error": "Could not retrieve leads export." }
```

**Note:** With Postgres configured, export is built from the database; otherwise from server-side Excel storage.

## 5. Admin API (JSON + exports)

These routes are for **internal dashboards**. They require authentication **only if** the server has **`ADMIN_API_KEY`** set in environment:

- Header: **`Authorization: Bearer <ADMIN_API_KEY>`**  
  **or**  
- Header: **`x-admin-key: <ADMIN_API_KEY>`**

If `ADMIN_API_KEY` is **not** set, the server allows `/admin/*` without credentials (**not recommended in production**).

### 5.1 List leads (paginated)

**`GET {API_BASE}/admin/leads`**

#### Query parameters

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `page` | integer | `1` | Page number (≥ 1). |
| `limit` | integer | `25` | Page size (1–100). |
| `q` | string | `""` | Search: name, email, phone, session_id, intent, aircraft, notes, budget (ILIKE). |
| `intent` | string | `""` | Filter by intent substring. |
| `bot` | string | (see below) | Chatbot slug; must be allowed for this deployment’s admin list. |

If `bot` is omitted, the server uses the deployment’s **default bot** (or first visible workspace). Invalid/disallowed `bot` → empty result set behavior (server may resolve to `null` internally for counts).

#### Response `200` — JSON

```json
{
  "items": [
    {
      "id": "uuid",
      "session_id": "string",
      "name": "",
      "email": "",
      "phone": "",
      "intent": "",
      "aircraft_type": "",
      "budget": "",
      "notes": "",
      "bot": "aeromaverick",
      "created_at": "2026-03-26T12:00:00.000Z"
    }
  ],
  "total": 100,
  "page": 1,
  "limit": 25,
  "pages": 4
}
```

#### Other statuses

| Status | Body |
|--------|------|
| `401` | `{ "error": "Unauthorized" }` if admin key required and missing/wrong |
| `503` | `{ "error": "Database not configured. …" }` |
| `500` | `{ "error": "Could not load leads." }` |

---

### 5.2 One lead by ID

**`GET {API_BASE}/admin/leads/:id?bot=<slug>`**

| Param | Location | Required |
|-------|----------|----------|
| `id` | Path | Yes — UUID |
| `bot` | Query | Effectively required for scoping; wrong/missing may `404` |

#### Response `200` — JSON

Single lead object (same fields as an element of `items` above).

#### Response `404` — JSON

```json
{ "error": "Not found" }
```

---

### 5.3 Bulk delete leads

**`POST {API_BASE}/admin/leads/delete?bot=<slug>`**  

**or** **`DELETE {API_BASE}/admin/leads?bot=<slug>`** (same handler)

#### Request body — JSON

```json
{
  "ids": ["uuid-1", "uuid-2"]
}
```

#### Response `200` — JSON

```json
{
  "deleted": 2,
  "bot": "aeromaverick"
}
```

#### Response `400` — JSON

```json
{ "error": "Invalid workspace" }
```

---

### 5.4 Export CSV

**`GET {API_BASE}/admin/leads/export.csv?bot=<slug>`**

Response `200`: `text/csv; charset=utf-8`, UTF-8 BOM prefixed, attachment filename `leads_<bot>.csv`.

Column order: `id`, `created_at`, `bot`, `session_id`, `name`, `email`, `phone`, `intent`, `aircraft_type`, `budget`, `notes`.

---

### 5.5 Export XLSX

**`GET {API_BASE}/admin/leads/export.xlsx?bot=<slug>`**

Response `200`: Excel attachment `leads_<bot>.xlsx`.

---

### 5.6 Stats

**`GET {API_BASE}/admin/stats?bot=<slug>&intent=<optional>`**

#### Response `200` — JSON

```json
{
  "leads_total": 42,
  "database": true,
  "bot": "aeromaverick"
}
```

If DB is off: `leads_total: 0`, `database: false`.

These are valid API/workspace slugs (server may restrict which appear in admin UI via env):

| Slug | Typical label |
|------|----------------|
| `aeromaverick` | AeroMaverick |

| `meddozer` | Meddozer |

## 7. Lead object shape (when `lead_saved` is true)

Fields are strings unless noted.

| Field | Description |
|-------|-------------|
| `name` | Contact name |
| `email` | Email (required for persistence when extracted from model) |
| `phone` | Phone |
| `intent` | e.g. `buy`, `sell`, `finance`, `charter`, `engine_stand`, `auction`, `general` |
| `aircraft_type` | Free text |
| `budget` | Free text |
| `notes` | Free text |
| `bot` | Slug stored with the lead |

The chat model emits a structured block that the server parses; the client usually only displays `reply` and optional confirmation when `lead_saved` is true.

## 8. Optional: hosted admin UI

If the same deployment serves static files: **`GET {API_BASE}/panel/`** — browser admin (not a JSON API). Your frontend integration normally uses **`/chat`** and **`/config`** only.

---

## 9. Quick checklist for a new frontend

1. **`GET /health`** — 200 OK.  
2. **`GET /config`** — read `defaultBot`, `chatbots`, `adminAuthRequired`.  
3. **`POST /chat`** — send `message`; store `session_id`; repeat with same `session_id`.  
4. If building an admin: add **`Authorization: Bearer …`** when `adminAuthRequired` is true.  
5. Use **`{API_BASE}`** with **no trailing slash** when concatenating paths (`/chat`, not `//chat`).

---

## 10. Environment (for DevOps, not browser)

Server must have **`OPENAI_API_KEY`** for chat. For DB-backed admin: **`DATABASE_URL`** or **`POSTGRES_URL`** or **`POSTGRES_PRISMA_URL`**. Optional: **`ADMIN_API_KEY`**, **`PUBLIC_API_URL`**, **`BOT_SLUG`**.  
Browser clients must **not** receive server-only secrets except optional **`VITE_ADMIN_API_KEY`** if you deliberately embed it in a static build (generally avoid).

This file is the contract for **`backend/server.js`** as of the repo version it ships with; after backend changes, re-read the code or regenerate this doc.
